# Homework #1 — Operate Claude Code on a Real Task

**Assigned after:** Session 2 (Hours 3–4)
**Due before:** Session 3
**Time budget:** 30–60 minutes

---

## The Assignment

Pick a small, concrete task from your current work — something you'd normally do by hand in a terminal. Not a toy problem. A real one.

Then complete it end-to-end using Claude Code.

---

## Requirements

**1. Use Claude Code to complete the task — not to assist while you do the typing.**
The goal is a full agentic pass: you describe the task, the agent plans and executes, you review and approve. No tab-completing your way through with the agent watching.

**2. Use planning mode at least once.**
Before executing a substantive step, ask Claude Code to write a plan. Review the plan before you let it proceed. Even if the plan is obvious, practice the habit.

**3. Try one short Auto Mode session.**
Switch to Auto Mode for part of the task. Let the agent make a few decisions without stopping. Note one moment where Auto Mode approved an action you'd have paused to review manually — or note if nothing surprised you.

---

## Reflection — Three Bullets

After the session, write three short bullets:

1. **One thing that worked better than expected**
2. **One thing that surprised you** (could be good, bad, or just unexpected)
3. **One question you want answered in Session 3**

Keep each bullet to two or three sentences. You are not writing an essay — you are capturing the thing that's freshest in your memory while it's still fresh.

---

## What to Bring to Session 3

Your three bullets. We'll surface a few at the start of Session 3 — 2–3 people will share briefly and we'll use them to open the session.

You don't need to prepare slides or a write-up. Just bring the bullets — and if you hit something interesting, be ready to describe what happened in two minutes.

---

## Choosing a Task

Good candidates for this homework:

- Running a report or data pull you normally script by hand
- Generating boilerplate for a new module, endpoint, or test suite
- Refactoring a function or file you've been meaning to clean up
- Scripting a repetitive git workflow (branch, commit, PR)
- Writing a small helper or wrapper script for a task your team does frequently

**Not recommended:** tasks that require access to production systems you don't have a safe way to sandbox, or tasks where a mistake would be hard to reverse. The point of this homework is to learn the tool, not to introduce risk.
